The workflow

From agent output to a page you can read

Nothing about this requires a new tool for the agent to learn. It writes plain markdown, the way it already does. The change is entirely on your side of the screen.

  1. Your agent, Claude Code, Cursor, Codex CLI, or anything that writes files, drops plan.md in the project folder.
  2. Open it in a browser tab: file:///…/plan.md. Skim renders it: headings, checklists, tables, code, mermaid diagrams. No need to bounce over to an editor's markdown preview pane.
  3. Leave the tab open. Skim watches the file and re-renders it in place, scroll position preserved, every time the agent rewrites it. No refresh key.
  4. Want to steer the agent? Copy for AI puts the current markdown on your clipboard, clean, in one click.

Made for agent files

Every agent convention, rendered

Frontmatter cards

Agents love YAML headers. Skim renders the block at the top of a file as a tidy key/value card instead of a wall of raw dashes.

Folder view

An agent's output directory becomes a navigable reading list. Jump from plan.md to report.md without leaving the tab.

Mermaid and KaTeX

Architecture diagrams and math render as diagrams and equations, not fenced code blocks left to rot in a corner of the repo.

Offline and private

A plan for your unreleased product never leaves the machine. No accounts, no network calls, nothing collected.

None of this needs configuration. Point Skim at the file, in a tab or in a folder, and these show up because the markdown asks for them, not because you flipped a setting.

Honest scope

Skim is a viewer, nothing more

Skim doesn't run agents, doesn't call any LLM, and collects nothing. It has no opinion about which agent you use, which model sits behind it, or what you ask it to do. It is the reading surface for tools you already run: the page a plan.md or a report.md becomes once the agent has written it, or while it's still writing. Everything it renders comes from a file already sitting on your disk.
